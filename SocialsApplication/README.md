# Docs Overview

This folder keeps the merge evidence and non-runtime documentation out of the application code.

- `audits/` records what was included, merged, deduplicated, or excluded.
- `manifests/` records upstream package manifests plus the final generated manifest and checksums.
- `licenses/` preserves upstream license texts and licensing notes discovered in the provided packages.
- `LICENSING.md` summarizes practical license handling for this generated package.
- `MERGE_AUDIT.md` explains how the three merged packages became `Socials Application`.
- `SECURITY_AUDIT.md` explains why risky or service-abuse code was not included in the runtime.
